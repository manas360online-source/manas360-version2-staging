7. Phase 6: Blueprint Automation

Navigate to: Setup → Blueprint → New Blueprint

Blueprint 1: Therapist Onboarding

Blueprint Name: Therapist Onboarding Pipeline
Department: Clinical + Legal & Contracts
Applies to: Tickets with tag "therapist_onboarding"

STATES (in order):
┌──────────┐    ┌──────────┐    ┌────────────┐    ┌──────────┐    ┌────────┐
│ Applied  │ →  │ Verified │ →  │ Contracted │ →  │ Training │ →  │ Active │
└──────────┘    └──────────┘    └────────────┘    └──────────┘    └────────┘

TRANSITIONS:
1. Applied → Verified
   Mandatory Fields: license_number, qualification, NMC_status
   Allowed Roles: TADM, BDE
   Custom Function: validate_therapist_credentials (Deluge)
   SLA: 48 hours from ticket creation

2. Verified → Contracted
   Mandatory Fields: contract_sent, contract_signed
   Allowed Roles: TADM, CCA
   Action: Trigger Zoho Sign template "MANS360_Therapist_ICA_v3"
   SLA: 7 days from verification

3. Contracted → Training  
   Mandatory Fields: training_enrolled, training_link_sent
   Allowed Roles: TADM
   Action: Send ZeptoMail training enrollment email
   SLA: 1 day from contract signing

4. Training → Active
   Mandatory Fields: quiz_score (must be >= 80), cert_generated
   Allowed Roles: TADM
   Before Transition: Check all 4 gates (license ✓, contract ✓, training ✓, bank ✓)
   Action: Update Supabase therapist status = 'active'
   Notification: ZeptoMail activation email + Twilio SMS

Blueprint 2: Clinical Escalation

Blueprint Name: Clinical Escalation (Crisis)
Department: Clinical
Applies to: Tickets with priority "Urgent" AND tag "crisis"

STATES:
┌───────────┐    ┌───────────────┐    ┌──────────┐    ┌──────────┐
│ Triggered │ →  │ PSR Assigned  │ →  │ In Care  │ →  │ Resolved │
└───────────┘    └───────────────┘    └──────────┘    └──────────┘

TRANSITIONS:
1. Triggered → PSR Assigned
   Auto-Assignment: Round-robin to available PSR agent
   SLA: 15 MINUTES (critical)
   Notification: Twilio SMS to assigned psychiatrist
   Escalation: If no response in 10min → auto-assign backup PSR

2. PSR Assigned → In Care
   Mandatory Fields: patient_contacted, risk_assessment
   Allowed Roles: PSR
   SLA: 1 hour from assignment

3. In Care → Resolved
   Mandatory Fields: outcome_notes, follow_up_scheduled
   Allowed Roles: PSR, TADM
   Action: Generate incident report, schedule 24h follow-up
   Notification: Update referring clinician (PSY)

Blueprint 3: Insurance Claims

Blueprint Name: Insurance Claim Processing
Department: Insurance + Finance
Applies to: Tickets with tag "insurance_claim"

STATES:
┌──────────┐    ┌────────┐    ┌─────────┐    ┌───────────┐    ┌────────┐
│ Submitted│ →  │ Verify │ →  │ Process │ →  │ Reimburse │ →  │ Closed │
└──────────┘    └────────┘    └─────────┘    └───────────┘    └────────┘

TRANSITIONS:
1. Submitted → Verify
   Allowed Roles: BDE, FIN
   Action: Validate claim against insurance partner API
   SLA: 24 hours

2. Verify → Process
   Allowed Roles: FIN, INS (own org claims)
   Mandatory: claim_amount, policy_number, verification_status
   SLA: 48 hours

3. Process → Reimburse
   Allowed Roles: FIN
   Action: Trigger PhonePe payout to patient (copay refund)
   SLA: 5 business days

4. Reimburse → Closed
   Allowed Roles: FIN, TADM
   Mandatory: payment_reference, patient_notified
   Action: ZeptoMail confirmation to patient + insurer

Department: Engineering
Condition: All new tickets
Assignment: Round Robin → DEV-01, DEV-02, DEV-03, DEV-04
Skip if: Agent is offline/on leave

// CLINICAL: Priority-based routing
Rule Name: Clinical Priority Router
Department: Clinical
Conditions:
  IF Priority == "Urgent" AND Tags contains "crisis"
    → Assign to PSR (round-robin: PSR-01, PSR-02)
  ELSE IF Tags contains "medication" OR Tags contains "prescription"
    → Assign to PSR (round-robin)
  ELSE
    → Assign to PSY (round-robin: PSY-01 to PSY-04)

// FINANCE: Direct assignment
Rule Name: Finance Assignment
Department: Finance
Assignment: Round Robin → FIN-01, FIN-02

9.2 Deluge: Auto-Tagging Incoming Tickets

// Workflow Rule: Auto-tag tickets based on subject/description keywords
// Setup → Workflow Rules → New Rule
// Trigger: On ticket creation (all departments)

subject = ticket.get("subject").toLowerCase();
description = ticket.get("description").toLowerCase();

tags = List();

// Clinical keywords
if (subject.contains("crisis") || description.contains("suicidal") || description.contains("self-harm"))
{
  tags.add("crisis");
  zoho.desk.updateTicket(ticket.get("id"), {"priority": "Urgent"});
}

if (subject.contains("phq") || description.contains("assessment"))
{
  tags.add("assessment");
}

// Payment keywords
if (subject.contains("payment failed") || description.contains("refund"))
{
  tags.add("payment_issue");
}

if (subject.contains("payout") || description.contains("wallet"))
{
  tags.add("therapist_payout");
}

// Onboarding keywords
if (subject.contains("onboarding") || description.contains("credential"))
{
  tags.add("therapist_onboarding");
}

// DPDPA keywords
if (subject.contains("delete my data") || description.contains("dpdpa") || description.contains("right to erasure"))
{
  tags.add("dpdpa_request");
  tags.add("compliance");
}

// Insurance keywords
if (subject.contains("claim") || description.contains("reimbursement"))
{
  tags.add("insurance_claim");
}

// Apply tags
if (tags.size() > 0)
{
  zoho.desk.updateTicket(ticket.get("id"), {"tags": tags});
}
