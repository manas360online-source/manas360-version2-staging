Lead Matching V2.0

Multi-dimensional matching engine. Upgrades single-variable matching to 14-dimension scoring with certification nudge, night-shift routing, AnytimeBuddy allocation, and corporate lead intelligence.

1

What's Changing: V1.0 → V2.0

Every limitation in V1 is now a multi-select dimension in V2

⚠️ V1.0 — Current (Single-Select)

  • Single specialization match (1 concern → 1 spec)
  • Single language match (patient picks 1, therapist has 1)
  • Single time-slot matching (morning/afternoon/evening)
  • Single location (city only)
  • No mode matching (assumed video)
  • No certification awareness
  • No corporate lead routing
  • No AnytimeBuddy allocation
  • No night-shift premium logic
  • No severity-weighted routing
  • No gender preference matching
  • No therapist velocity scoring
  • No waitlist intelligence
  • No re-match continuity for returning patients
  • 6 dimensions, max 100 points
  • Static weights — same formula for all leads

✅ V2.0 — Patch (Multi-Dimensional)

  • Multi-spec: patient 3+ concerns ↔ therapist 9 categories
  • Multi-language: any combination of 14 Indian languages
  • Multi-shift: 3 shift windows × 7 days with day-pill overlap
  • Multi-location: 15 cities + Tier 2/3 wildcard
  • Multi-mode: Video / Phone / In-person / Chat
  • Certification-aware: nudge to certify if corporate lead matches
  • Corporate lead routing with dedicated counselor matching
  • AnytimeBuddy opt-in → instant 15-min call allocation
  • Night-shift premium: +10% fee, routed to willing therapists
  • Severity-weighted: PHQ-9/GAD-7 → therapist tier routing
  • Gender preference: patient requests, therapist serves
  • Response velocity: fast responders get better leads
  • Waitlist intelligence: queue if best match is full
  • Continuity scoring: returning patients → same therapist bonus
  • 14 dimensions, max 200 points
  • Dynamic weights — context-adaptive per lead type

Context-Adaptive Weight Multipliers

Different lead types shift the weight distribution dynamically
💡 Wisdom Silo: V1 used static weights for all leads. V2 recognizes that a corporate EAP request values certification and location over severity, while a crisis escalation values severity and availability over budget. The same 14 dimensions, weighted differently per context.
Python # Context multipliers — applied BEFORE summing WEIGHT_PROFILES = { "standard_patient": { # Default — balanced weights (1.0x = no change) "specialization": 1.0, "language": 1.0, "availability": 1.0, "budget": 1.0, "mode": 1.0, "location": 1.0, "severity": 1.0, "service": 1.0, "certification": 1.0, }, "corporate_eap": { # Corporate leads: certification + location matter most "specialization": 0.8, "language": 1.2, "availability": 1.0, "budget": 0.5, # corporate pays, budget less relevant "mode": 1.5, # in-person vs virtual critical "location": 2.0, # must be in same city for on-site "certification": 2.0, # certified therapists preferred "service": 1.5, # EAP toggle must be on }, "crisis_escalation": { # Crisis: severity + availability + speed trump everything "severity": 2.0, "availability": 2.0, "velocity": 2.0, "service": 1.5, # crisis toggle must be on "budget": 0.0, # crisis = free, budget irrelevant "certification": 0.5, "location": 0.3, }, "anytime_buddy": { # Quick 15-min calls: availability + service toggle "availability": 2.0, "service": 2.0, "language": 1.5, "velocity": 1.5, "specialization": 0.5, # How toggles from the profile power real-time routing decisions
Night Shift Router # Night Shift (10 PM — 6 AM) — Premium Routing def route_night_patient(patient, timestamp): # Hard filter: ONLY therapists with night_shift_on = True pool = therapists.filter( night_shift_on=True, shifts__contains="night", status="available" ) # Apply premium pricing for t in pool: t.effective_rate = t.session_rate * 1.10 # +10% night premium # Patient notification notify_patient( "Night sessions carry a 10% premium. " "Your therapist receives this premium for late-night availability." ) # Match against filtered pool using V2 algorithm matches = match_v2(patient, pool, context="night_shift") return matches
AnytimeBuddy Router # AnytimeBuddy — Quick 15-min Support Call def route_anytime_buddy(patient): # Hard filter: ONLY therapists with anytime_buddy_on = True # AND currently within their active shift window now = datetime.now() pool = therapists.filter( anytime_buddy_on=True, is_in_active_shift(now)=True, current_session=None, # not in a session right now ) # Sort by: language match → response velocity → availability # AnytimeBuddy is SPEED-FIRST matching matches = match_v2(patient, pool, context="anytime_buddy") if len(matches) == 0: # Fallback: route to Dr. Meera AI chatbot return fallback_to_ai(patient, "dr_meera") # Auto-connect to top match (no lead purchase needed) # Therapist gets ₹150 flat fee per AnytimeBuddy call return auto_connect(matches[0], patient, duration=15)

Waitlist Intelligence

🧠 Wisdom Silo: When best match is full, don't lose the patient
🔄 V1 problem: if top-3 matched therapists are fully booked, patient sees "no therapists available" and churns. V2 introduces intelligent waitlisting with ETA notifications and interim AI support.
Waitlist def handle_full_matches(patient, matches): available = [m for m in matches if m.has_open_slots()] waitlisted = [m for m in matches if not m.has_open_slots()] if available: return show_available(available) # normal flow # ALL top matches are full if waitlisted: best = waitlisted[0] eta = best.next_open_slot() # e.g., "3 days" return { "action": "waitlist_offer", "message": f"Your best match ({best.name}, {best.score}% match) " f"opens up in ~{eta}. Want to join waitlist?", "options": [ "✅ Join waitlist (notify me)", "👥 Show next-best available therapists", "🤖 Chat with Dr. Meera AI while waiting", "📞 Try AnytimeBuddy for quick support now", ] }
