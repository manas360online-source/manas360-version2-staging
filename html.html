Zoho Flow Automation - Visual Blueprint (A to Z)

This is the exact structure to implement in Zoho Flow for MANAS360 WhatsApp automation. Start from webhook, branch by event, branch again by role/type when needed, then send via Watti.

Trigger: Webhook Decision 1: Event Decision 2: Role / Clinical Type Action: Build Variables Action: Watti API Decision 3: Success / Retry

B

Total Decision Blocks in Zoho Flow: 1 main event decision + 3 sub-decisions (role, payment role/domain, clinicalType) + 1 send result decision.

A to Z Build Order (Zoho Blocks)

  1. Create Flow: MANAS360 - Event Router
  2. Add Trigger: Incoming Webhook
  3. Capture fields: event, source, timestamp, data.*
  4. Add Decision block: event
  5. Create branch: USER_REGISTERED
  6. Create sub-decision: data.role
  7. Map role to welcome template
  8. Create branch: SESSION_BOOKED
  9. Add patient booking template action
  10. Add provider booking template action
  11. Create branch: PAYMENT_SUCCESS
  12. Map payment success variables
  13. Create branch: PAYMENT_FAILED
  14. Add sub-decision for patient/provider
  15. Map failed-payment template variables
  16. Create branch: SESSION_FOLLOWUP
  17. Map follow-up variables
  18. Create branch: CLINICAL_EVENT
  19. Add sub-decision: data.clinicalType
  20. Map clinical type to template
  21. Add DEFAULT branch logging
  22. Add common HTTP action config (Watti)
  23. Add response decision: success/failure
  24. Add retry/backoff on failure
  25. Add delivery logs
  26. Test all 6 main events with sample payloads
Important: Do not hardcode template variables inside one generic block. For each branch, map only that template's required variables to avoid runtime template mismatch errors.

Watti Template Variables for Links (Jitsi, Prescription, Report)

These fields should be present in webhook data and mapped inside Set Variables before HTTP call.

Use case Webhook Field Example Value Typical Template Variable
Session join link (Jitsi) Trigger-Webhook.jitsiLink https://meet.jit.si/manas360-room-123 join_link
Prescription download link Trigger-Webhook.prescriptionUrl https://app.manas360.com/prescriptions/abc/download prescription_link
Clinical report link Trigger-Webhook.reportUrl https://app.manas360.com/reports/r_123 report_link
Provider profile / follow-up link Trigger-Webhook.providerDashboardUrl https://app.manas360.com/provider/dashboard cta_link
Keep role values consistent in one style only: either uppercase role values everywhere or lowercase everywhere. Mixed values (example: PATIENT in one branch and patient in another) can break decision matching.
